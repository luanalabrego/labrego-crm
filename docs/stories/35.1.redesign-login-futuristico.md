## Story

Como **usuário do Voxium**, quero uma tela de login com design moderno, futurístico e sofisticado, que transmita profissionalismo e confiança na plataforma.

---

## Problem

A tela de login atual é funcional mas básica: um card branco centralizado com inputs simples sobre fundo gradiente suave (`from-primary-50 to-slate-100`). Não transmite a identidade visual sofisticada que a marca Voxium deseja.

---

## Acceptance Criteria

### AC1: Design moderno com elementos futurísticos
- **Given** o usuário acessa `/login`
- **When** a página carrega
- **Then** exibe um layout de tela cheia com:
  - Background escuro (slate-900/950) com elementos visuais sofisticados
  - Efeitos de glassmorphism (backdrop-blur, bg-white/5) no card de login
  - Gradientes sutis com as cores da marca (cyan #13DEFC → green #09B00F)
  - Elementos decorativos (grid pattern, glow effects, orbs ou partículas)

### AC2: Branding proeminente
- **Given** a tela de login
- **When** o usuário visualiza
- **Then** o logo "Voxium" está em destaque com estilo gradiente
- **And** há um tagline ou descrição curta da plataforma
- **And** a identidade visual é consistente com o restante do app

### AC3: Layout split-screen ou hero
- **Given** a tela de login em desktop
- **When** exibida em tela cheia
- **Then** usa layout split-screen (lado esquerdo: hero/branding, lado direito: form)
  OU layout centralizado com hero section acima do form
- **And** o formulário tem inputs com estilo dark/glass
- **And** botões com gradiente da marca

### AC4: Responsivo e acessível
- **Given** a tela de login em mobile
- **When** exibida em tela pequena
- **Then** adapta para layout single-column
- **And** inputs e botões têm tamanho adequado para touch
- **And** contraste mínimo WCAG AA mantido

### AC5: Funcionalidade preservada
- **Given** o novo design
- **When** o usuário interage
- **Then** login com email/senha funciona identicamente
- **And** "Esqueceu a senha?" funciona identicamente
- **And** mensagens de erro são exibidas com estilo atualizado
- **And** estados de loading mantidos

---

## Scope

### IN Scope
- Redesign visual completo da página de login
- Efeitos CSS (glassmorphism, gradientes, animações sutis)
- Layout responsivo (desktop + mobile)
- Preservar toda lógica funcional existente

### OUT of Scope
- Novos métodos de autenticação (Google, GitHub, etc.)
- Página de registro/signup
- Alteração na lógica do Firebase Auth
- Animações complexas com bibliotecas externas

## Dev Notes

### Página atual (login/page.tsx)
- Lógica: `signInWithEmailAndPassword()`, `sendPasswordResetEmail()`
- States: email, password, error, loading, showReset, resetEmail, resetLoading
- UI: Card branco, fundo gradiente claro, inputs arredondados

### Referências de design
- **Glassmorphism**: `backdrop-blur-xl bg-white/5 border border-white/10`
- **Glow effects**: `shadow-[0_0_40px_rgba(19,222,252,0.15)]`
- **Grid pattern**: CSS background-image com grid sutil
- **Floating orbs**: divs absolutas com blur e gradiente

### Cores da marca
- Primary: `#13DEFC` (cyan)
- Accent: `#09B00F` (green)
- Background: `slate-900`, `slate-950`
