## Story

Como **usuário do CRM**, quero importar contatos a partir de arquivos **.xlsx** (Excel) além de CSV, para ter mais flexibilidade no processo de importação.

---

## Problem

Atualmente, o fluxo principal de importação de contatos em `/contatos` (linhas 424-520) só aceita CSV (lê como texto, split por vírgula). Porém, a biblioteca `xlsx` já está importada no arquivo (`import * as XLSX from 'xlsx'` — linha 12), e o upload de sócios/partners (linhas 613-688) já usa XLSX corretamente. O fluxo principal precisa ser unificado para aceitar ambos os formatos.

---

## Acceptance Criteria

### AC1: Aceitar arquivos .xlsx e .xls no input de importação
- **Given** o modal de importação de contatos
- **When** o usuário clica em "Importar"
- **Then** o input de arquivo aceita `.csv`, `.xlsx` e `.xls`
- **And** o atributo `accept` inclui `.csv,.xlsx,.xls,application/vnd.openxmlformats-officedocument.spreadsheetml.sheet,application/vnd.ms-excel`

### AC2: Parsing correto de arquivos Excel
- **Given** um arquivo .xlsx selecionado
- **When** o arquivo é carregado
- **Then** o sistema usa `XLSX.read()` para parsear o workbook
- **And** usa `XLSX.utils.sheet_to_json()` para extrair dados da primeira planilha
- **And** aplica o mesmo mapeamento de campos (nome→name, telefone→phone, etc.)

### AC3: Preview funciona para ambos os formatos
- **Given** um arquivo .xlsx ou .csv selecionado
- **When** o preview é gerado
- **Then** mostra as primeiras 5 linhas com os campos mapeados
- **And** o comportamento é idêntico ao CSV atual

### AC4: Importação funciona para ambos os formatos
- **Given** o preview confirmado pelo usuário
- **When** o usuário clica em confirmar importação
- **Then** todos os contatos são criados no Firestore com os campos corretos
- **And** campos obrigatórios (name, phone) são validados igualmente

## Dev Notes

### Padrão existente para reutilizar
O `parsePartnersData()` (linhas 613-688) já implementa detecção CSV vs Excel:
```typescript
const XLSX = await import('xlsx')
const workbook = XLSX.read(data, { type: 'array' })
const worksheet = workbook.Sheets[workbook.SheetNames[0]]
const rows = XLSX.utils.sheet_to_json(worksheet, { defval: '' })
```

### Estratégia de implementação
1. Na `handleImportFileChange()`, detectar extensão do arquivo
2. Se `.xlsx`/`.xls`: usar `FileReader.readAsArrayBuffer()` + `XLSX.read()`
3. Se `.csv`: manter lógica atual de `readAsText()`
4. Unificar output em formato `{ headers: string[], rows: string[][] }`
5. Manter mapeamento e validação existentes
